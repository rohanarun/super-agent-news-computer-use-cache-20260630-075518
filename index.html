What is a computer-use cache?

Direct answer: A computer-use cache stores the successful actions an AI agent took while operating a real browser or desktop, then replays those actions for future runs of the same workflow.

Instead of paying a model to re-plan clicks, typing, scrolling, and navigation every time, the cached action trace executes deterministically until something changes.

This mirrors emerging primitives in agent infrastructure that cache tool calls or scripts to reduce repeated model spend and improve reliability, as seen in open-source and browser-use systems.

How Super’s computer-use cache works

1) First run: full agent

Super observes the screen, plans steps, and operates the computer using visual actions (clicks, typing, scrolling). This is similar to the computer-use loops described in model tool docs.

2) Cache the successful actions

When the task completes, Super saves the action sequence and context as a reusable memory for that workflow.

3) Repeat runs: replay

On the next run, Super executes the cached actions directly. No re-planning unless the page changes.

4) Auto-heal when needed

If the UI changes, Super can fall back to a live agent run, then update the cache with the new successful path.
